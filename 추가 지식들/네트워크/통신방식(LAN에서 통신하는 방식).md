- 유니캐스트: 서버와 클라이언트 간의 1:1 통신 방식
	- 통신을 통해 데이터를 수신한 쪽은 수신한 데이터의 MAC 주소와 자신의 MAC 주소가 동일하면 자신을 목적지로 온 것이 맞으므로 수신하지만, 동일하지 않다면 데이터를 수신하지 않고 버림
- 브로드캐스트: 출발지는 하나지만 통신장비에 연결된 전부가 목적지인 통신 방식
	- 브로드캐스트의 주소는 FF-FF-FF-FF-FF-FF로 미리 설정되어 있음, 통신장비가 브로드캐스트의 주소를 받으면 자신과 연결되어 있는 모든 포트로 해당 데이터를 보냄
- 멀티캐스트: 출발지는 하나지만 목적지가 3개 이상인 경우(전부에 보내면 브로드 캐스트가 됨)
	- 근데 거의 안 씀ㅋㅋ, 왠지는 몰루


### LAN에서 실제 데이터를 전송받는 경우 ###
1. 나한테 온 프레임의 목적지 주소가 내 주소와 같은 경우 (유니캐스트)
2. 브로드캐스트 주소로 나한테 프레임이 전달딘 경우 (브로드캐스트)
3. 멀티캐스트 주소로 나한테 왔고 해당 멀티캐스트 주소가 내가 속한 그룹인 경우 (멀티캐스트)
4. 1~3 경우를 제외하면 나에게 도착한 프레임의 목적지가 내가 아니므로 NIC는 도착한 프레임을 수신하지 않고 그냥 버림
