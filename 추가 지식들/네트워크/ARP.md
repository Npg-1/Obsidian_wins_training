# ARP #
LAN에서 통신을 할 때 송신할 때 보낸 데이터가 제대로 도착하려면 MAC 주소가 필요함, IP주소를 통해서 컴퓨터 앞까지 갈 수는 있어도 랜카드(NIC)가 IP주소가 아닌 MAC 주소만을 수신하기 때문임 따라서 IP 주소를 MAC 주소로 바꿔줄 필요가 있음

1. 상대방에게 데이터를 보내려고함
2. 내 컴퓨터의 MAC 주소를 적어두는 주소부(ARP 테이블)에 상대방의 MAC 주소를 확인함
3. 없다면 현재 네트워크(사설망)에 연결되어 있는 모든 컴퓨터에 상대방의 IP를 보내고 응답으로 MAC주소를 기다림(브로드캐스트)
4. 상대방 컴퓨터가 MAC주소를 보냈다면 해당 주소를 가지고 데이터를 보냄
5. MAC주소는 ARP 테이블에 기록함

\* cmd에서 arp -a를 하면 내 컴퓨터의  ARP 테이블을 확인할 수 있음

## 특징 및 주의사항##
- **동일 네트워크(서브넷) 내에서만 작동:** ARP는 2계층 브로드캐스트를 사용하기 때문에, 라우터를 넘어서 외부 인터넷망까지 전달되지는 않습니다. (외부망으로 나갈 때는 게이트웨이의 MAC 주소를 찾습니다.)
- **ARP Spoofing (보안 이슈):** 누군가 중간에서 "내가 그 IP 주인이야!"라고 거짓말을 해서 데이터를 가로채는 공격이 있을 수 있습니다.
