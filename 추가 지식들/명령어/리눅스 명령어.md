## cat ##

## cd ##
- 지정된 경로로 현재 경로를 이동
	- 절대 경로
		- 절대경로로 지정하는 경우 우분투(리눅스) 에서는 C:\ 경로를 인식하지 못하기 때문에 윈도우와는 경로 표현 방식이 다름
		- 윈두우에서는 드라이브 문자(C:\\) 로 시작하는 것 대신에 /mnt/아래에 적음
		- (바탕화면으로 이동하기 예시)
			- "cd C:\Users\WINS_24\Desktop"         (Windows)
			- "cd /mnt/c/Users/WINS_24/Desktop"      (Linux)
	- 상대 경로
		- '/': 그냥 cd / 처럼 하면 최상위 루트로 이동
		- '.': 현재 디렉토리
		- '..': 상위 디렉토리
		- ex)
			- cd /my_app -> /가 맨앞에 오면 최상위 디렉토리를 뜻하므로 윈도우즈로 치면 C -> my_app로 이동하는 거임
			- cd ../Documents -> 상위 디렉토리로 올라갔다가 Documents로 이동
			- cd Desktop -> 현재 디렉토리의 하위 디렉토리 Desktop으로 이동
			- cd ./Desktop -> 이것도 현재 디렉토리에서 Desktop으로 가라는 거니까 cd 에서는 똑같은 의미임  
				\*하지만 다른 명령어에서는 의미가 달라질 수 있음  ([[#configure|./configure]] 참고)

## configure ##
- ./configure --prefix=/usr/local/apache2 --enable-so --enable-ssl --enable-rewrite
- 압축을 푼 소스 코드를 가지고 실제 프로그램을 생성하기 전 옵션을 지정하는 환경 설정 단계 명령어임
- cd 에서는 ./ 를 쓰면 현재 폴더의 특정 폴더로 이동하라는 의미라서 크게 의미가 없지만 configure와 같은 명령어에서는 ./를 쓰면 현재 폴더가 실행파일임을 명시하는 거여서 실행의 의미를 가지게됨
	- ./configure: 현재 폴더의 configure라는 실행 파일을 실행하라는 의미
	- --prefix=/usr/local/apache2: 아파치가 설치될 경로
	- (기타 옵션들은 커엇!)

## ps ##
- 현재 터미널에서 실행시킨 프로세스를 간략하게 보여줌
- PID(프로세스 ID), TTY(일단 몰라도 됨), TIME(실행 시간), CMD(해당 프로세스가 실행 될 때의 명령어 이름 or 실행 파일의 경로)
- 실행중인 게 따로 없을 때는 보통 bash와 ps가 뜸
	- bash: 터미널에 사용자가 명령어를 치면 해당 명령어가 올바른지 확인하고 컴퓨터에 전달하는 프로그램으로 터미널이 실행중일 때 우리는 bash 위에서 작업을 하고 있기 때문에 뜸
	- ps(Process Status): 사용자가 명령어를 입력하면 그 순간에 ps가 스냅샷을 찍음, 따라서 ps를 쳤을 때 ps가 실행되어서 실행 프로세스에 뜨게됨(?)

	- ps -e(Everyone): 모든 프로세스를 표시함, ps는 터미널에서 실행중인 프로세스를 보여주지만, ps -e를 하게 되면 **컴퓨터**에서 실행중인 모든 프로세스를 보여줌
	  
	- ps -f(Full format): 부모 프로세스 ID, 시작 시간 등 상세한 정보 표시
		- ps -ef: e, f를 붙여서 모든 프로세스를 상세히 표시함, (ps -fe와 같이 f, e 순서를 바꿔도 상관 없음)
		  
		- ps -ef | grep httpd: ps(Process Status) 현재 프로세스의 목록을 보여줌
			- ps -ef: 모든 프로세스를 상세히 표시
				- | grep httpd: 해당 목록들 중에서 httpd(아파치)라는 글자가 섞인 것만을 표시
	
	- ps -ef | grep less: 전체 시스템에서 실행한 프로세스를 출력
	  필터없이 전체 검색같은 느낌적인 느낌

## ss ##
- Linux 시스템에서 소켓(네트워크 소켓)과 관련된 정보를 표시하는 명령어
- ss -lnpt | grep httpd
	- l(listenning): 현재 연결된 상태가 아니라, 외부의 접속을 대기 중인 소켓만 표시(서버 전용)
	- n(numeric): 서비스 이름을 주소(http)가 아닌 숫자로 표시(속도가 빠름)
	- p(processes): 이 통로를 어떤 프로그램이 쓰고 있는지 표시
	- t(tcp): TCP 방식의 연결만 표시(웹 서버는 대부분 TCP를 사용함)
	- grep httpd: 넘겨받은 텍스트 중에서 httpd라는 글자가 포함된 줄만 표시


## sudo ##
- sudo(Super User DO): 관리자 권한으로 실행, 일반 명령어 앞에 sudo를 붙여주기만 하면 됨
	- sudo (경로/파일) start:  지정한 파일을 실행하는 명령어
		- ex) sudo /usr/local/apache1/bin/apachectl start: apache1 서버 열기
			- 비슷한 걸로 stop와 restart도 뒤에서 start만 바꿔주면 됨

	- sudo apt update
		- 리눅스 실행환경이 설치할 수 있는 최신 소프트웨어들의 목록을 업데이트 하는 명령어
		- 이 명령어를 통해 apt를 최신 목록으로 갱신해서 다운받을 것을 알 수 있음
			- apt(Advanced Package Tool): 우분투의 패키지 매니저로, 스마트폰의 앱스토어 같은 것

	- sudo pkill httpd


## tar ##
- tar (옵션) (파일명): 리눅스에서 가장 많이 쓰이는 압축 포맷인 .tar 또는 .tar.gz 파일을 묶거나 푸는 명령어
- 옵션들
	- -x(Extract): 압축 해제
	- -c(Create): 압축해서 새로운 압축 파일 생성
	- -v(Verbose): 과정 표시
	- -f(File): 파일 지정
