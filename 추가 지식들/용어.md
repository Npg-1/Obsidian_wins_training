## 확장자 ##
- msi: Windows의 소프트웨어 인스톨러, Windows OS에서는 디폴트로 exe와 거의 같음




## 도메인(Domain) ##
도메인 또는 도메인 네임이라 불리며 넓은 의미에서는 외우거나 식별하기 어려운 IP 주소(예: 240.10.20.1)를 exaple.com 처럼 기억하기 쉬운 이름으로 바꿔주는 것을 말한다


## QoS(Quality of Service) ##
네트워크 데이터 전송 품질을 보장하기 위해 데이터 흐름에 우선순위를 부여해서 데이터 전송을 관리하는 기술


## 옥텟(Octet) ##
컴퓨팅에서는 8비트(또는 바이트)를 의미함 예시로는 1011 1110 . 1010 1000 . 0000 0000 . 1001 0100 와 같은 2진수가 있으면 이 2진수는 4개의 옥텟으로 구성되어 있다고 한다.


## 호스트(Host) ##
일반적으로 호스트는 네트워크의 노드 중에서 IP 주소를 가지고 어플리케이션을 실행하는 종단장치를 뜻함
	ex) 데스크탑, 구글 서버, 스마트폰


## 제로 데이 공격 ##
특정 소프트웨어의 아직까지 공표되지 않은, 혹은 공표되었지만 아직까지 패치되지 않은 보안 취약점을 이용한 해킹


## MTU ##
MTU(Maximum Transmission Unit)는 한 번에 보낼 수 있는 '**최대 데이터 덩어리의 크기**'를 말함


## 프레임 & 패킷  ##
*여기서는 대략적인 구분의 용도로서의 뜻만 설명함*
프레임과 패킷은 서로 혼용해서 사용하기도 하지만 엄격하게는 서로 다른 것임
- 프레임:  패킷에 2계층 프로토콜 헤더(예: 이더넷)이 붙어있는 것
- 패킷 :  4계층에서 내려온 데이터에 3계층의 프로토콜(예: IP)가 붙어있는 것
	-> 즉 패킷(3계층)에 2계층 헤더가 붙으면 프레임(2계층)이 됨


## 트래픽 & 대역폭 ##
- **트래픽**: 네트워크상에서 흐르는 데이터의 양 트래픽이 높다는 건 데이터 전송량이 많다는 것이고, 트래픽이 낮다면 데이터 전송량이 낮다는 것임
- **대역폭**: 한정된 시간 동안 전송될 수 있는 최대 데이터 양을 의미하는 것
- 트래픽과 대역폭의 관계: 트래픽이 낮다면 대역폭이 어느정도 낮아도 문제가 없지만 트래픽이 높을 때 대역폭도 같이 높지 않다면 데이터 전송에 지연이 발생할 수 있음


## 트래픽 미러링  ##
네트워크 스위치나 라우터를 통과하는 데이터의 흐름을 중단시키지 않고 똑같은 복사본을 만들어 다른 곳으로 보내주는 기술
복사본에서 해킹신호가 있는지 확인하는 등의 용도로 사용함


## 엔드포인트(Endpoint)
컴퓨터 네트워크에 연결되어 정보를 주고받는 모든 기기(컴퓨터, 사므트폰, IoT 장비 등)


## Iaas/PaaS/Saas ##
- 클라우드 제공 업체가 사용자들에게 **어디까지 관리해 주는가**를 분류해 놓은 것
	1) Application : 응용 프로그램
	2) Data : 응용 프로그램의 서비스를 제공하기 위해 축적하는 데이터
	3) Runtime : 프로그램 코드 실행 도구
	4) Middleware : 운영체제 및 응용프로그램 연결용 소프트웨어
	5) OS : 운영 체제
	6) Virtualization : 가상화 기능
	7) Server : 전송 장치
	8) Storage : 데이터 저장 장치
	9) Networking : 외부 통신을 위한 네트워크 시스템
	   
	- **Iaas**(Intrastructure as a Service)
		- 인프라 레벨의 제공 및 관리하는 서비스로 1~5 까지는 사용자가 관리하지만 6~9까지는 클라우드 컴퓨팅 서비스를 제공받음
		- 클라우드에서 가상 서버나 온라인 스토리지 등을 이용하기 때문에 서버나 스토리지, 네트워크 관리할 필요가 없어서 비용절감이 됨
			- ex) AWS, Microsoft Azure, Google Cloud
		
	- **Paas**(Platform as a Service)
		- 앱 구축 및 개발 환경 플랫폼을 제공 및 관리하는 서비스로 1~2까지는 사용자가 관리, 3~9는 서비스를 제공받음
		- Iaas에 비해서 제공받는 서비스가 더 많이 때문에 소프트웨어 업데이트나 하드웨어 유지보수 및 관리는 하지 않아도 되어서 좋음 -> 사요자는 응용 프로그램 서비스외에는 신경 안 써도 됨
		- PaaS는 프로그램만 개발만 하면 되어서 장점이 있지만 기본적으로 제공받은 플랫폼 위에서 동작하는 것이므로 다른 플랫폼으로 이동하기 까다로움
			- ex) AWS Elastic Beanstalk, Heroku, Red Hat OpenShift
		
	- **Saas**(Software as a Service)
		- 걍 다 제공함, IT 인프라의 모든 요소, 모든 클라우드 서비스를 다 제공받을 수 있음 사용자는 아무런 설치나 서비스 구축을 안 하고 제공되는 서비스를 이용하기만 하면 됨
		- 근데 그만큼 비용은 비쌈
			- ex) Snowflake, Dropbox, Salesforce, Google Apps, Red Hat Insights
