# 프로토콜의 개념 #

## 프로토콜(Protocol) ##
- 개체간의 효율적인 통신을 위한 규칙
- 데이터의 포맷, 전송과 관련된 규칙을 정의함
- 서로 연결된 네트워크에는 같은 규칙을 사용하여야 통신 가능
- 현재 인터넷 기반의 네트워크는 TCP/IP 프로토콜을 기반으로 구성되어 있음

# 주요 네트워크 프로토콜 #

## IP 헤더  ##

![[Pasted image 20251218162307.png]]
- 네트워크 계층의 핵심 프로토콜로 전송 계층으로부터 세그먼트를 받아서 인접한 네트워크가 요구하는 크기의 패킷으로 분할 전송
- 경로를 제공하지만, 흐름 제어나 에러 제어는 미지원 
- IP 헤더의 크기는 20~60 바이트임
	- Ver(3Bit): IPv4와 IPv6를 구분하기 위한 정보
	- HeaderLENgth(4Bit): 전체 패킷의 길이를 표시
	- Identification/Flags/Fragmentation offset: [[#Data Fragmentation (데이터 단편화)|패킷 단편화]]와 관련된 정보
	- Time to Live: [[용어#프레임 & 패킷|패킷]]의 수명
	- Protocol Identifier: 패킷 내에 포함된 상위 프로토콜 종류
		- ?
	- Checksum: 헤더 오류 검출용 데이터
	- Source Address: 송신자 IP 주소 
	- Destination Address: 수신자 IP 주소 

#### Data Fragmentation (데이터 단편화) ####
- 각 장비들의 [[용어|MTU]] 값이 다르므로, 네트워크 내에서 사용되는 패킷의 크기에 따라 분할하여 도착지에서 재조합하는 과정

## TCP 헤더 ##
![[Pasted image 20251218170045.png]]

- Source Port: 송신자 포트
- Destination Port: 수신자 포트
- Sequence Number: 송신하는 데이터의 첫 번째 바이트 위치(?)
- Acknowledgement Number: 수신하기를 기대하는 다음 바이트 번호 (?)
- HLEN: 헤더 길이
- Flags: TCP 회선 및 데이터 관리 제어 기능을 위한 플래그
- Window Size: 흐름 제어를 위한 버퍼 여유 용량 크기 통보
- Urgent Pointer: URG Flag가 설정될 경우 현재 Seq Num부터 긴급데이터까지의 byte offset (현재는 거의 안 씀)

# 기타 #

### 포트주소 ###
- 동일한 IP 주소를 가진 시스템 내에서 프로세스를 구분하기 위한 주소
- 0~2<sup>16</sup>(=65536)-1 범위의 번호를 사용 -> 16비트 주소
- 0~2<sup>10</sup>(=1024)-1: Well-known port(=System port), 잘 알려진 서비스/서버 들이 사용
- 1024~49151: Registered port(=User port), 응용프로그램들 사용, 서버나 클라이언트가 사용할 때도 있음
- 49151~65535: Dynamic port, 접속을 하는 용도로 주로 사용, 주로 클라이언트들이 사용
