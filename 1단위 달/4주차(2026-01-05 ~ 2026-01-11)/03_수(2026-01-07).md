



HA(High Availabiliy): 고가용성, 이중화라고 부르기도함, %로 나타냄, 긴 시간동안 운영이 가능한 시스템의 살태를 말함
가용성은 99.9%와 같이 퍼센트로 나타내고, 서비스가 중단되지 않고 운영될 수 있는 능력임, 총 운영시간 중 시스템의 가동시간 비율을 나타냄

로드밸런싱: 여러 서버에 걸쳐 들어오는 네트워크 트래픽(부하)을 고르게 분산시켜 특정 서버에 과부하가 걸리는 것을 막고, 시스템의 안정성, 가용성, 성능을 최적화하는 기술 및 장치

ARP: P 주소(논리적 주소)를 MAC 주소(물리적 주소)로 변환하여 로컬 네트워크(LAN) 내 장치들이 서로 통신할 수 있게 해주는 네트워크 프로토콜

Failover: 연결 끊는 것


Active-Active면 장비 2개에 VIP도 2개가 있어야함
둘 중에 하나가 죽으면 한 쪽이 들고 있던 VIP를 다른 쪽으로 옮겨가는 건 Active-StandBy랑 같음


GARP(Gratuitous ARP): Active 장비가 변경되었을 때 MAC 주소를 변경시키기 위한 ARP
Active 장비가 죽으면 Backup 장비에서 MAC 주소를 브로드캐스트 해서 수신한 장비들은 MAC Table을 갱신함


스위치끼리는 패킷이 넘어갈 수 있지만 보안장비(방화벽) 끼리는 패킷이 안 넘어감
방화벽 사이의 인터링크(?)를 통해서 세션동기화, 설정동기화, 로그동기화 기능만 수행하고 패킷은 전송 안 됨



이중화는 가상의 IP를 공유를 했을 때 1번이 Active 2번이 백업 장비면 IP를 3개를 받음, (Active-Standby)
VIP(Virtual IP)를 가지고 있는 쪽과 L2 스위치가 통신을 하는데 1번 장비가 죽으면 죽기 전에 VIP를 2번으로 옮겨서 L2가 1번 대신에 2번의 VIP와 통신을 해서 운영이 끊기지 않게 할 수 있음

20문제 중 5문제는 찝어줌

20문항은 랜덤
- Q1)방화벽은 무엇인가
	- A1)네트워크에서 불법 침입을 차단하기 위해 일반적으로 네트워크 구조의 최상단에 위치하며, 접근 시도를 1차적 제어, 통제(허용/거부) 함으로써 내부 네트워크를 보호하는 역할을 수행
- Q2) TCP/UDP 차이점
	- A2) TCP는 통신에서 신뢰성을 보장할 수 있지만 UDP는 그냥 쏴버림, 
	- 받았어? 진짜 받았어?? 보낸다? 나 보낸다??? -> TCP => 연결 지향적, 신뢰성 높음, 상대적으로 느림
	- 몰라레후 일단 쏨 받든말단 내 알바 아님 -> UDP      => 비연결형, 신뢰성 낮음, 빠른 것이다
- Q3) NGFW 보안 정책의 매칭 4가지 순서
	- 보안정책 우선 순위: 패킷 유입 -> 방향성 결정 -> 우선순위 -> 매칭
- Q4) 리눅스에는 tcpdump라는 게 있고 NGFW에는 캡쳐가 무엇인가
	- tcpdump는 리눅스의 네트워크를 타고 흐르는 데이터 패킷을 실시간으로 엿보고 기록하는 도구로 NGFW의 캡쳐와 거의 같음
- Q5) 방화벽이 세션을 기억한다는 게 어떤 의미인가
	- 방화벽에는 상태 기반 검사(Stateful Inspection)이라는 기술이 있음
	- 방화벽 내부에 상태 테이블이 있고 내가 사용자가 외부의 웹사이트에 접속하려고 하면 방화벽은 사용자가 빠져나갈 때 "사용자가 네이버에 접속을 요청함! 네이버에서 응답이 올 듯함!"이라고 메모를 해두고 나중에 응답이 오면 확인하고 방화벽에서 따로 확인하는 과정을 거치지 않아서 효율적임
	- 방화벽이 세션을 기억하면 ,TCP 연결 과정에서 발생하는 SYN, ACK 같은 신호들이 정상적인 순서로 오고 가는지 감시할 수 있어서 순서에 맞지 않는 패킷(ACK만 온다거나)이 들어오면 순서에 맞지 않으므로 '세션에 맞지 않는 패킷'으로 판단해 차단할 수 있음
- Q6) Wins 방화벽의 웹필터, 어플리케이션 보안 등의 트래커 기능이 포함됨
	- 그냥 글타고, 진짜 웹필터, 어플리케이션, IPS 등을 볼 수 있는 트래커가 있음! (근데 메뉴에 항목은 없네)
- Q7) 브릿지와 라우터(L3) 및 방화벽의 관계
	- A7) 
	- 브릿지는 2계층에서 서로 다른 세그먼트를 연결하는 역할을 함, 사설망 내부에서 서로 연결하는 역할만 하므로 외부의 네트워크와는 분리되어 있음, MAC 주소를 보고 데이터를 보낼 곳을 결정함, 최근에는 Switch가 대신하고 있음
	- 브릿지는 3계층에서 서로 다른 네트워크(LAN과 LAN, LAN과 인터넷 등)을 연결하는 역할을함, 
	- 통신 순서는 인터넷 -> 라우터 -> 방화벽 -> 브릿지 -> 컴퓨터
