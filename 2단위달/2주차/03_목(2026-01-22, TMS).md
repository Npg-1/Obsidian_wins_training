
TMS-PLUS란?
자사 보안장비의 이벤트와 네트워크 정보를 수집/분석/대응하고 **연동장비를 관리**하는 통합보안관리시스템
- NGFW, DDoS, IPS, APT 등의 로그를 다 받을 수 있음


상황판에서 장비의 현재 상태, 로그들을 볼 수 있음

TMS는 데이터 로그를 많이 보관하기 때문에 하드디스크와 메모리 용량이 많이 큼

Snort 배포는 -rule을 배포해서 각 장비에 뿌리는 기능을 가지고 있음

## 2.2 모니터링 ##
연동된 센서와 등록된 객체에서 전송된 이벤트와 네트워크 트래픽을 제공하는 메뉴

- 상황판 - 실시간 서버현황, 이벤트/트래픽 현황, 분석 등 레이아웃 별 표시
- 이벤트 모니터링 - 실시간 탐지되는 이벤트에 대한 정보를 확인할 수 있는 메뉴
	  - 필터 뷰: 기본 필터/고급 필터 적용으로 조건에 해당되는 이벤트만 모니터링
	  - 데이터 연관분석: 탐지된 이벤트에 대한 연관분석 정보를 확인
	  - IP/PORT 차단: 탐지된 이벤트에 대한 IP/PORT 차단
  - 트래픽: 연동된 장비 별 실시간 트래픽을 확인하는 메뉴
  - 실시간 트래픽: 연동된 장비 별 실시간 INLINE 트래픽을 확인하는 메뉴(트래픽이랑 차이점이 INLINE 밖에 없는 건가)
  - 세션: 연동된 방화벽 센서의 세션 모니터링 확인하는 메뉴

- 모니터링
	- 상황판: 수집한 데이터에 대하여 관리자가 설정한 정책에 의해 탐지된 보안위반 사건과 리소스 상태정보를 가시적으로 제공
	- 이벤트: 연동된 센서와 등록된 객체에서 탐지한 이벤트와 트래픽을 실시간으로 제공
	- 실시간 트래픽: 연동된 모든 센서에서 수집되고 있는 실시간 트래픽에 대해 TCP, UDP, ETC, DROP 으로 구분하여 데이터를 제공
	- 리소스 상태정보: TMS 서버 및 연동된 센서 장비의 리소스 정보를 실시간으로 제공
	- 세션: 연동된 방화벽 센서의 세션 상태를 실시간을 제공


- 분석
	- 연관 분석: 탐지 이벤트와 트래픽에 대해 지난 주와 현재를 비교하여 추이 그래프 및 분석 정보 제공
	- 프로파일링 분석: 프로파일링 정책에 의해 탐지된 이벤트와 트래픽에 대해 24시간 프로파일링 분석 정보 제공
	  (프로파일링 정책은 임계치를 설정하여 임계치 위반 시 탐지가 되는 구조)
	- 상관 분석: 관리자가 설정한 상관분석 정책에 의해 탐지된 탐지 이벤트에 대한 24시간 상관 분석 정보 제공
	  (상관분석 정책은 탐지 이벤트에 대한 정책 템플릿을 생성하여 탐지하는 구조)

- 통계 - 이벤트 순위
	- 공격명, 공격IP, 대상IP, 대상 PORT, 공격국가, 대상국가, 센서의 7개 기준 이벤트 순위 제공

- 조회
	- 이벤트: 연동된 센서에서 탐지한 세션 로그 및 이벤트 로그 조회
	- 트래픽: 연동된 센서에서 탐지한 트래픽에 대한 로그 조회
	- 리소스 상태 정보: TMS-Plus Server와 이에 연동한 센서에 대한 리소스 상태 정보 내역 조회
	- 프로파일링 경보: 등록된 프로파일링 정책에 의해 발생한 경보 이력 조회
	- 상관 경보: 관리자가 내린 상관 분석 정책에 의해 발생한 경보 이력 조회
	- 리소스 임계치 경보: 등록된 리소스 임계치 정책 값을 초과하거나 시스템 이상으로 경고 발생한 경우에 대한 이력 조회
	- NMS: 연동된 센서에서 발생된 NMS 이벤트 로그 조회
	- 룰 배포 이력: TMS-Plus 에서 내린 룰 배포 이력 조회
