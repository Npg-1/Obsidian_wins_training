
TMS-PLUS란?
자사 보안장비의 이벤트와 네트워크 정보를 수집/분석/대응하고 **연동장비를 관리**하는 통합보안관리시스템
- NGFW, DDoS, IPS, APT 등의 로그를 다 받을 수 있음


상황판에서 장비의 현재 상태, 로그들을 볼 수 있음

TMS는 데이터 로그를 많이 보관하기 때문에 하드디스크와 메모리 용량이 많이 큼

Snort 배포는 -rule을 배포해서 각 장비에 뿌리는 기능을 가지고 있음

# 2. TMS-PLUS 기능 #

## 2.1 로그인 ##
SniperLauncher로 로그인
https://tms-plus:3000


## 2.2 모니터링 ##
연동된 센서와 등록된 객체에서 전송된 이벤트와 네트워크 트래픽을 제공하는 메뉴

- 상황판 - 실시간 서버현황, 이벤트/트래픽 현황, 분석 등 레이아웃 별 표시
- 이벤트 모니터링 - 실시간 탐지되는 이벤트에 대한 정보를 확인할 수 있는 메뉴
	  - 필터 뷰: 기본 필터/고급 필터 적용으로 조건에 해당되는 이벤트만 모니터링
	  - 데이터 연관분석: 탐지된 이벤트에 대한 연관분석 정보를 확인
	  - IP/PORT 차단: 탐지된 이벤트에 대한 IP/PORT 차단
  - 트래픽: 연동된 장비 별 실시간 트래픽을 확인하는 메뉴
  - 실시간 트래픽: 연동된 장비 별 실시간 INLINE 트래픽을 확인하는 메뉴(트래픽이랑 차이점이 INLINE 밖에 없는 건가)
  - 세션: 연동된 방화벽 센서의 세션 모니터링 확인하는 메뉴





































- 상황판 - 실시간 서버현황, 이벤트/트래픽 현황, 분석 등 레이아웃 별 표시
	- 서버 현황: TMS 자산관리 등록된 장비 상태를 표시
	- 상관 분석: 상관 분석 정책 설정 후 탐지 건수 표시
	- 프로파일링: 프로파일링 정책 설정 후 탐지 건수 표시
	- 국가별 탐지/차단: 수집된 이벤트 로그 중 국가 코드를 기반으로 탐지/차단 정보 표시
	- 이벤트 현황: 금일, 전일 이벤트 건수 및 시도 건수 비교를 통한 탐지 이벤트 현황 분석
	- 트래픽 현황: 금일, 전일 bps/pps 비교
- 이벤트 모니터링 - 실시간 탐지되는 이벤트에 대한 정보를 확인할 수 있는 메뉴
	- 센서/객체: 등록한  센서/객체 정보를 표시
	- 탐지건수 추이(5초 이벤트): 실시간 추이/24시간 추이 선택 표시
	- 유해 트래픽 bps 추이(5초 트래픽): 실시간 추이/24시간 추이 선택 표시
	- 탐지/시도 Top5
		- 실시간 탐지이벤트 대상 (5초 통계)
		- 탐지/시도 집계 기준 선택 가능
		- 공격 IP/대상 IP/서비스/공격명 집계 항목 출력
	- bps/pps Top5
		- 실시간 탐지이벤트 대상 (5초 통계)
		- bps/pps 집계 기준 선택 가능
		- 공격 IP/대상 IP/서비스/공격명 집계 항목 출력
	- 실시간 탐지
		- 실시간 탐지되는 이벤트 표시
		- 100, 300, 500, 1000, 5000 출력건수 선택
		- 고급필터 기능 제공

























#### 모니터링 ####
- 상황판 수집한 데이터에 대하여 관리자가 설정한 정책에 의해 탐지된 보안위반 사건과 리소스 상태정보를 가시적으로 제공하는 메인화면
	- 서버 현황: 장비에 문제가 생기면 빨간 느낌표가 뜸, 해당
- 이벤트 모니터링: 실시간 탐지되는 이벤트에 대한 정보를 확인할 수 있는 메뉴
	- 서버 현황: 장비에 문제가 생기면 빨간 느낌표가 뜸, 해당
- 실시간 트래픽: 실시간으로 트래픽을 볼 수 있음, 장비에 공격이 들어오는지 등을 볼 수 있고, 
- 리소스: 연동된 센서의 리소스 상태를 확인하는 메뉴
- NMS: 특정 고객사에서 본인들이 쓰는 기준이라서 TMS에서도 그 기준을 맞추기 위해 따로 갖추고 있음, 중요하진 않음
- 차단: 차단되고 있는 로그 등을 볼 수 있음. IP차단(블랙리스트)도 여기서 따로 볼 수도 있어서 차단로그는 여기서 보는 게 편하고 좋음
	- IP/PORT를 차단하는 기능도 포함하고 있어서 


#### 2.4 통계 ####

- 이벤트 추이
- 트래픽 순위
- 트래픽 추이
- 차단 순위
- 차단 추이
