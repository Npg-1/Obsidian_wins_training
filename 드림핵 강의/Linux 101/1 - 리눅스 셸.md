# 셸(Shell)이란? #

운영체제는 크게 유저 레벨과 커널 레벨로 구분됨. 유저는 이 중에서 유저 레벨을 사용할 수 있고 커널 레벨은 일반 사용자는 조작할 수 없음. 따라서 운영체제를 온전히 사용하기 위해서는 커널과 유저 사이를 이어주는 프로그램인 **셸(Shell)** 이 필요하게됨

셸은 영어로 껍질이라는 의미를 가지며 컴퓨터공학에서는 사용자가 운영체제와 상호작용 할 수 있는 인터페이스를 제공하는 소프트웨어를 의미함

사용자가 셸에 명령을 입력하면 셸은 이를 해석하여 동작을 수행함. 이때 셸은 커널과 사용자를 이어주는 인터페이스 역할을 하며 마치 운영체제의 핵심을 감싸는 껍데기(Shell) 처럼 보이기 때문에 이러한 명칭이 붙음

![[Pasted image 20260130092854.png]]

셸은 크게 GUI 셸과 CLI 셸로 나눌 수 있는데 일반적으로 사용하는 Windows와 macOS의 검색창이 대표적인 GUI 셸이고, Windows의 cmd와 macOS의 터미널 등이 대표적인 CLI 셸임

GUI셸의 경우 사용자들이 직관적으로 기능을 파악하고 사용하는 것에 장점이 있지만 글개픽 작업이 CPU나 GPU를 상대적으로 많이 소모하기 때문에 속도가 느리다는 단점이 있음. 단변 CLI셸의 경우 아이콘과 같은 이미지를 보여주지 않기 때문에 상대적으로 비직관적이고, 원활한 사용을 위해서는 셸에서 사용하는 명령어들을 암기해야 한다는 단점이 있으나 실행속도가 빠르고 정밀한 적업을 수행하기 편함.

프로그래밍 언어로 비교하자면 GUI 셸은 스크래치로, CLI 셸은 C언어로 비유할 수 있음. 스크래치는 눈에 보이는 블럭을 배치해 프로그래밍하기 때문에 각 명령이 하는 역할을 직관적으로 이해할 수 있지만 복잡한 명령들을 스크래치로 구현하기는 어려움. 하지만 C는 명령어와 문법을 일일이 학습해야하고 사용하기 어려우나 익히기만 하면 운영체제에 복잡한 명령을 내릴 수 있음

일반적으로 리눅스에서 말하는 셸은 CLI 셸을 뜻함

# 터미널(Terminal) #

터미널은 CLI 셸을 GUI 셸 위에서 사용할 수 있도록 하는 프로그램임. 

기본 셸은 **bash**를 사용하지만 이 외에도 **sh, zsh**와 같이 다양한 셸이 존재하고 각기 고유한 특징이 있음. 


---
# Sh #

Bourne Shell(sh)은 유닉스 버전 7의 기본 셸이며 최초로 사용되던 Thompson Shell, 그리고 PWB Shell을 대체하기 위해 등장함.
약자가 셸을 대표하는 sh인 이유는 셸의 기본이 되었던 Thompson Shell이 sh라는 이름을 사용해서 Bourne Shell은 이를 대체하는 셸이었기 때문

sh는 아래와 같은 기능을 지원함
- 기본적인 프로그램 실행
- 기본적인 입출력
- 환경 변수 설정
- 셸 프로그래밍(**if**, **while** 등) 지원


---
# bash #

**Bourne-Again Shell(bash)** 은 **sh**를 대체하기 위해 Brian Fox가 작성한 유닉스 셸로 GNU 운영체제와 리눅스, macOS 등의 기본 셸로 탑재되었으며 이후 DOS와 Microsoft Windows로도 이식된 매우 대중적인 셸임. 일반 사용자들이 기본적으로 로그인하면 볼 수 있는 셸이 bash임

**bash**는 **sh**의 기능에 더해 아래의 기능을 추가로 지원함
- 명령 히스토리(이전에 실행했던 명령을 저장해둠) 지원
- Tab 자동 완성 지원
- **bash** 로그인 시 아래 시작 파일 자동 실행
	1. /etc/profile 이 존재하면 읽은 후 실행
	2. ~/.bash_profile, ~/.bash_login, ~/.profile 을 순차적으로 탐색해 존재하면 읽은 후 실행
- **bash** 종료 시 ~/.bash_logout 자동 실행
- 셸에서 **bash** 실행 시 /etc/bash.bashrc 와 ~/.bashrc 가 존재하면 자동 실행

#### 실행과 로그인의 차이 ####
*로그인은 사용자가 아이디와 비밀번호 등을 입력해 셸을 접속하는 상황을 의미함. 주로 **secure shell(ssh)** 접속같은 경우를 뜻함*
*반면 실행은 콘솔 창에서 bash 명령어를 실행하거나 스크립트 내부에서 bash를 호출하는 것을 의미함*

# ash & dash #

**Almquist Shell(ash)** 과 **Debian Almquist Shell(dash)** 은 **bash**보다 경량화된 유닉스 셸임. FreeBSD, NetBSD, 그리고 몇 리눅스 배포판의 기본 셸로 설치됨

ash와 dash는 bash보다 가볍고, 라이브러리 의존성이 상대적으로 적음. 이런 이유로 다음과 같은 경우에 주로 사용함
- 플로피 디스크에 셸을 설치하는 상황
- 루트 사용자의 셸
- /bin/sh (sh) 대체용


---
# zsh #

**Z Shell(ash)** 은 Paul Falstad가 최초로 작성한 유닉스 셸임. Bourne Shell을 기반으로, bash, ksh 등의 다양한 셸의 장점을 모아 만든 확장형 셸으로 커스텀하기 용이하며 설치도 간편하다는 특징이 있음. macOS는 zsh를 기본 셸로 사용함

## 키워드 ##
- ### Shell ###
	- 사용자와 커널을 이어주는 인터페이스를 제공하는 프로그램으로 커널을 감싸는 껍데기 같다고 해서 Shell 이라는 이름이 붙게 됨
- ### Bourne Shell (sh) ###
	- 유닉스 버전 7의 기본 셸이며, 기본적인 프로그램, IO 명령 실행, 셸 프로그래밍 등을 지원함
- ### Bourne-again Shell (bash) ###
	- sh를 대체하기 위해 Brian Fox가 작성한 유닉스 셸로 sh의 기능에 더해 명령 히스토리, tab 자동 완성, 실행 및 종료 시 기본 스크립트 실행 등을 지원함
- ### Almquist shell (ash)/Debian Almquist shell (dash) ###
	- bash보다 경량화된 유닉스 셸
- ### Z Shell (zsh) ###
	- Paul Falstad가 최초로 작성한 유닉스 셸임. Bourne Shell을 기반으로, bash, ksh 등의 다양한 셸의 장점을 모아 만든 확장형 셸임
