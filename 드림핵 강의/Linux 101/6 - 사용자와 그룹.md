# 리눅스 권한 관리 방식 #

## 유저와 그룹 ##

리눅스는 **유저(u, user)** 와 **그룹(g, group)** 을 가짐. 유저는 온라인 사이트의 계정과 유사한 개념이며 ==한 유저가 여러 개의 유저 권한을 가질 수 없음==. 그룹은 특정한 권한을 가진 사용자의 집단을 표현하기 위한 것이며 한 유저는 여러 개의 그룹에 속할 수 있음

유저 정보는 `/etc/passwd`에 저장됨. `cat /etc/passwd`로 확인해보면 아래와 같음
![[Pasted image 20260304230357.png]]

각 문자열이 가리키는 건 아래와 같음

`[이름]:[암호]:[UID]:[GID]:[추가정보]:[홈 디렉토리]:[기본 셸]`

| 정보     | 상태                                     |
| ------ | -------------------------------------- |
| 이름     | 유저의 이름                                 |
| 암호     | 사용자의 암호, x인 경우 `/etc/shadow`에 따로 암호 저장 |
| UID    | 사용자의 id                                |
| GID    | 그룹 id                                  |
| 추가 정보  | 사용자의 추가 정보 저장                          |
| 홈 디렉토리 | 사용자의 홈 디렉토리 경로                         |
| 기본 쉘   | 해당 계정으로 로그인 시 사용할 셸                    |

`/etc/shadow`는 계정의 암호와 만료 기간 등을 저장함. root 계정만 `cat /etc/shadow`로 읽을 수 있음. root 계정은 관리자 권한 계정으로 리눅스 운영체제에서 가질 수 있는 최고 권한을 가짐. 운영체제 설정 변경이나 계정 생성, 삭제와 같이 일반 유저들이 수행할 수 없는 작업들을 할 수 있음

`sudo`는 root로 명령어를 실행할 수 있도록 하는 명령어로 일단 지금은 어떤 명령을 root 권한으로 실행하고 싶을 때 사용하는 명령어라고 생각하면 됨

아래는 `sudo`를 사용해서 `/etc/passwd`를 본 것 (`/etc/passwd`는 일반 유저는 권한 부족으로 못 봄)

![[Pasted image 20260305225206.png]]

각 문자열이 가리키는 건 아래와 같음

`[이름]:[암호화된 암호 값]:[수정 일자]:[최소 사용 기간]:[최대 사용 기간]:[경고 기간]:[만기일]:[차단 기간]:[reserved]`

| 정보        | 상태                                                                                                                                                        |
| --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 이름        | 유저의 이름                                                                                                                                                    |
| 암호화된 암호 값 | `$[id]$[salt]$[hash]`의 형태이며 각각<br>- id: 해싱 알고리즘<br>- salt: 해시 시 salt값<br>- hash: password와 salt를 id에 할당하는 해싱 알고리즘으로 해싱 시 결과<br>`*`는 어떤 password에도 대응되지 않음 |
| 수정 일자     | 유닉스 시간(1970년 1월 1일) 기준으로 패스워드가 수정된 일자.<br>0이라면 다음 로그인 시 패스워드를 변경해야함                                                                                       |
| 최소 사용 기간  | 패스워드 변경 일자로부터 최소 사용 기간 이후 다시 변경 가능. 0이라면 언제든 변경 가능                                                                                                        |
| 최대 사용 기간  | 패스워드의 유효 기간. 99999라면 만료되지 않음                                                                                                                              |
| 경고 기간     | 패스워드 만료 전 사용자에게 미리 경고해주는 기간                                                                                                                               |
| 만기일       | 패스워드 사용 기간이 만료된 후 실제로 계정이 사용 불가능하기까지의 기간                                                                                                                  |
| 차단 기간     | 유닉스 시간(1970년 1월 1일) 기준으로 패스워드가 만료될(된) 일자                                                                                                                  |
| reserved  | 아직 사용하지 않음                                                                                                                                                |


그룹 정보는 `/etc/group`에 저장됨 아래는 `cat /etc/group`으로 확인한 것
![[Pasted image 20260305230154.png]]

각 문자열이 가리키는 건 아래와 같음
`[이름]:[암호]:[GID]:[멤버]`

| 정보  | 상태                                     |
| --- | -------------------------------------- |
| 이름  | 그룹의 이름                                 |
| 암호  | 그룹의 암호, x인 경우 `/etc/gshadow`에 따로 암호 저장 |
| GID | 그룹 id                                  |
| 멤버  | 해당 그룹에 포함되는 유저 이름들                     |

`/etc/gshadow`는 그룹 패스워드와 몇 가지 정보를 저장함. root만 `cat /etc/gshadow`로 읽을 수 있음

![[Pasted image 20260306122621.png]]

각 문자열이 가리키는 건 아래와 같음
`[이름]:[암호]:[관리자]:[멤버]`

| 정보  | 상태                                |
| --- | --------------------------------- |
| 이름  | 그룹의 이름                            |
| 암호  | 그룹의 암호, `*`는 어떤 password도 대응되지 않음 |
| 관리자 | 그룹의 관리자                           |
| 멤버  | 해당 그룹에 포함되는 유저 이름들                |

## 리눅스 파일 권한 ##

리눅스는 파일의 권한을 **소유자(u, user)**, **그룹(g, group)**, **일반 사용자(o, other)** 주체 각각에 대해 **읽기(r, read)**, **쓰기(w, write)**, **실행하기(x, execute)** 3가지 권한을 부여하며, 이를 종합하면 총 9가지의 조합이 나옴. 각 권한은 모두 독립적이며 동시에 가질 수 있음. 주로 권한을 숫자료 표현하며 r == 4, w == 2, x == 1에 대응됨. 각 숫자를 2진수로 표현하면 `0b100, 0b010, 0b001`이고 각 비트를 **비트플래그**로 생각하여 사용함. 비트플래그란 각 비트를 하나의 플래그로 생각하는 방식으로, 만약 어떤 파일의 소유자 권한이 `0b101`이라면 **0b100 | 0b001**로 생각하여 **r, x**권한을 가지고 있다고 판정하는 방식

| 권한         | 상태                |
| ---------- | ----------------- |
| -r-------- | 소유자에게 읽기 권한       |
| --w------- | 소유자에게 쓰기 권한       |
| ---x------ | 소유자에게 실행 권한       |
| ----r----- | 그룹에게 읽기 권한        |
| -----w---- | 그룹에게 쓰기 권한        |
| ------x--- | 그룹에게 실행 권한        |
| -------r-- | 다른 모든 사용자에게 읽기 권한 |
| --------w- | 다른 모든 사용자에게 쓰기 권한 |
| ---------x | 다른 모든 사용자에게 실행 권한 |

## 유저 생성 (useradd, adduser) ##

useradd로 새로운 유저를 생성할 수 있음. 계정 생성은 root만 가능함

`useraddd [옵션] [유저 이름]` 혹은 `useradd [유저 이름] [옵션]`으로 사용하며 아래와 같이 사용함

`useradd`와 달리 `adduser`는 다양한 설정들을 편리하게 수행할 수 있음. 그래서 일반적으로는 `adduser`를 사용함

## 유저 삭제(userdel, deluser) ##

userdel로 유저를 제거할 수도 있음

`userdel [옵션] [유저 이름]` 또는 `deluser [옵션] [유저 이름]`로 사용하며 옵션은 아래와 같음

| --- | ------------------ |
| -f  | 강제로 사용자를 삭제        |
| -r  | 사용자의 홈 디렉토리도 함께 삭제 |
