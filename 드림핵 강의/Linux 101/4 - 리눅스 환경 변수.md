# 환경 변수란? #
## 환경변수 ##

**환경 변수(Environment Variable)** 는 컴퓨터에서 실행 중인 프로세스의 동작 방식에 영향을 미치는 값이며 사용자가 임의로 정의할 수 있음. 환경 변수는 일반적으로 **키-값(Key-Value)** 쌍의 형태로 구성되며 시스템 및 프로그램의 설정이나 동작에 필요한 정보를 전달하는 데 사용됨

특정 디렉토리 경로, 사용자 이름, 시스템 환경 설정 또는 응용 프로그램이 참조할 API 키 등을 환경 변수로 지정할 수 있음

리눅스에서 사용하는 대표적인 환경 변수 키 값들 몇 가지

| 환경 변수      | 역할                                     |
| ---------- | -------------------------------------- |
| \***PATH** | 실행파일 탐색 경로 집합 (가장 자주 사용되는 환경 변수 중 하나임) |
| HOME       | 사용자의 홈 디렉토리                            |
| LANG       | 터미널에서 기본적으로 지원할 언어                     |
| PWD        | 사용자의 현재 작업 디렉토리 경로                     |
| USER       | 사용자 이름                                 |
| SHELL      | 로그인 셸 종류                               |
| UID        | 사용자의 UID                               |

터미널에서 `sh`만 입력해도 `/bin/sh`가 실행되는 이유는 PATH 환경 변수에 `/bin`이 포함되어 있기 때문임. 만약 터미널에 명령어가 입력되는 경우 (ex. `cat`) 셸은 PATH의 맨 앞 폴더부터 시작해 해당 폴더에 사용자가 요청한 프로그램이 있는지 확인하고 있다면 해당 프로그램을 실행함

# 환경 변수 확인 및 변경 #

## 환경 변수 확인(env) ##

`env`는 **env**iron의 약자로 현재 환경 변수를 출력하거나 새로운 환경 변수를 설정한 후 해당 환경 변수로 프로그램을 실행하는데 사용함

`env [옵션] [NAME=VALUE(환경 변수)] [명령어]`으로 사용하고 옵션은 아래와 같음

- -i: 현재 환경 변수 설정을 모두 무시하고 지정한 환경 변수 값을 따름
- -u: 지정된 변수의 설정 값 해제

`env`만 사용할 경우 설정된 환경 변수를 모두 보여줌

`env -i [env1] [env2] ... [env n] [파일 이름]` 을 사용할 경우 설정한 환경변수만 사용함

## 키워드 ##
- ### 명령어 ###
	- `ps`: 현재 운영체제에서 동작하고 있는 프로세스의 정보를 확인

## 키워드 ##
- ### 환경 변수 ###
	- 컴퓨터에서 실행 중인 프로세스의 동작 방식에 영향을 미치는 값이며 일반적으로 `Key-Value` 쌍의 형태로 구성됨
- ### 주요 명령어 ###
	- `env`: 환경 변수를 확인할 수 있음
	- `export`: 환경 변수를 추가할 수 있음
	- `unset`: 환경 변수를 제거할 수 있음
