프록시
- 클라이언트와 서버 사이에서 통신을 할 때, 중간에서 '중계 역할을 수행하는 서버'를 뜻함. 즉, 클라이언트와 서버 간에는 직접적으로 통신하는 것이 아닌 중계 역할을 하는 '프록시 서버'를 거쳐서 서로간의 요청-응답을 수행함

![[Pasted image 20260211101634.png]]


**클라이언트와 서버 사이에 프록시 서버가 존재하면, 클라이언트가 호출하는 엔드포인트는 프록시 서버가 되는 것인가?**
- 프록시를 사용하면 클라이언트는 프록시 서버로 요청을 보내야함. 프록시 서버는 이러한 요청을 받아 실제 서버로 전달하고, 서버의 응답을 다시 클라이언트로 전달함
- 이를 통해서 클라이언트는 직접 서버와 통신하지 않고 프록시 서버를 통해 간접적으로 통신하게 됨
- 단, DNS를 이용하면 프록시 서버의 역할을 보다 간편하게 수행할 수 있음. DNS를 통해 도메인 이름을 프록시 서버의 IP 주소로 매핑하면, 클라이언트는 별도의 설정 변경 없이도 프록시 서버를 통해 통신할 수 있음
